# DEV_PLAN.md — Полный план разработки курса SL-CLAW

> Внутренний документ команды разработки. Структура: тарифные границы (📍 НАЧАЛО / 📍 КОНЕЦ), уроки внутри, сценарии записи. Команда работает по этому документу — каждый урок имеет owner-а, сценарий и критерий «готово».

## 0. Обзор тарифов

| Тариф | Цена | Уроков | Хронометраж | ETA продаж | Owner |
|---|---|---|---|---|---|
| **MINI** | **$25** | 7 | ~90 мин | 2026-05-18 | CTO + продюсер |
| **CORE** | **$300** | ~83 | ~12 часов | 2026-06-15 | CTO + методолог |
| **PRO** | **$600** | ~30 | ~7 часов | 2026-07-06 | CTO + sales-coach |
| **IMPL** | **$2 000** | SOP (не уроки) | — | 2026-07-13 | CTO + аккаунт |

**Каскад**: каждый старший тариф включает всё из младших + новый блок. Студент платит только разницу при апсейле.

**Логика монетизации**: уроки записаны так, что в **последнем уроке каждого тарифа** студент чётко понимает лимит данного тарифа и видит конкретную ценность следующего. Это **не маркетинг** — это **сама структура контента**.

## Структура сценария одного урока (шаблон)

```
### Урок [код]. [Название]
**Длительность:** N мин
**Cтатус:** 🟡/⚪/🔴/✅
**Owner:** имя
**Зависимости:** какие файлы, какой код, какие предыдущие уроки

**Цель урока:** одна фраза, что студент должен понять/сделать после.

**Что показываем на экране** (cinematography):
- 0:00–0:30 — кадр 1 (например: face cam + слайд)
- 0:30–2:00 — кадр 2 (терминал + скринкаст)
- ...

**Что говорим** (script):
> «Текст голосом. Прямой текст того, что говорит ведущий…»
- Ключевые тезисы
- Возможные импровизации

**Что делает студент** (action):
- Конкретная команда / клик / правка кода

**Результат после урока** (deliverable):
- Что у студента есть на руках
- Какой checkpoint выполнен

**Связь со следующим уроком / апселлом**:
- Как урок ведёт дальше
- Если последний в тарифе — какой триггер апсейла
```
