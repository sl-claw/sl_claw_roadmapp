## 🆕 КЛЮЧЕВАЯ КОНЦЕПЦИЯ PRO — Niche Marketplace

> Главная новая фича PRO ($600): после оплаты в личном кабинете открывается **витрина готовых ниш**. Каждую можно одной кнопкой импортировать в свой инстанс sl-claw. Можно также выгружать свои настроенные ниши как `.niche` файл — для передачи клиенту, копии между серверами или (в будущем) продажи в маркетплейсе.

### Что такое ниша как продукт

Ниша = упакованный набор:
- `persona.py` — TenantPersona (имя, компания, бизнес, ценности, цель, тон)
- `catalog.json` — структурированный каталог товаров/услуг
- `objections.md` — типовые возражения (50+ вариантов)
- `spin_questions.md` — SPIN-вопросы под нишу
- `meddpicc.md` — критерии квалификации B2B (если применимо)
- `cases.md` — реальные кейсы успешных закрытий
- `tone.md` — стилевые особенности
- `cta_phrases.md` — готовые CTA на каждой стадии
- `integrations/*.py` — нишевые интеграции (например, Bitrix24 для риэлторов)
- `prompts/` — кастомные промпт-фрагменты
- `media/` — фото товаров, видео-кейсы (опционально)
- `meta.yaml` — метаданные (имя, версия, автор, лицензия, требуемые модули)

### Как студент устанавливает нишу в свой инстанс

**Способ 1: команда в Telegram-боте**
```
/install niche-realestate
```
Бот делает запрос на access-service с lux-token студента → проверка тарифа → если PRO+ → backend генерирует уникальный `.niche` файл с временной ссылкой → бот скачивает и распаковывает в `app/modules/niches/realestate/` → перезагружает приложение → ниша активна.

**Способ 2: ручная установка через веб-кабинет**
- Кнопка [Скачать .niche] → файл `niche-realestate-v1.2.3.niche`
- В CLI: `sl-claw niche install ~/Downloads/niche-realestate-v1.2.3.niche`
- Скрипт распаковывает → проверяет meta.yaml → копирует файлы → запускает `bulk_train` для md → перезагружает агент

**Способ 3: через Coolify (для производственной среды)**
- В Environment Variables → `INSTALL_NICHES=realestate,construction`
- При деплое контейнер сам скачивает ниши с access-service по lux-token
- Ниши поднимаются автоматически

### Как студент выгружает свою настроенную нишу

```bash
sl-claw niche export my-tuned-realestate
# → создаёт my-tuned-realestate-v1.0.0.niche в текущей папке
```

Что собирается:
- TenantPersona из БД
- Каталог из pgvector с вектороми **(опционально — без, если хотим сэкономить размер)**
- Все факты из manual / facts / pdf_facts
- Кастомные prompts
- Все интеграции из `app/modules/niches/<slug>/integrations/`

Затем студент может:
- Передать `.niche` своему клиенту (он установит у себя)
- Сохранить копию для другого клиента в той же нише
- (В будущем) Опубликовать в Niche Marketplace и продавать

### Niche Marketplace — Phase 2 (Q4 2026)

Расширение концепции — позволить **PRO-юзерам публиковать свои ниши в маркетплейсе** и зарабатывать %.

Workflow:

2. Загружает на token.lux-promo.com/marketplace/niches/upload
3. Модерация (24-48 часов)

5. Маркетплейс берёт 30% комиссии, автор получает 70%

Это превращает SL-CLAW в платформу с UGC и долгосрочным income для топ-юзеров.

## Технические требования к Niche Marketplace (для команды разработки)

### Backend `sl-claw/marketplace-service`
- FastAPI + Postgres + S3
- Endpoints:
  - `POST /niches/upload` (для авторов, требует PRO+)
  - `GET /niches/list` (публичный каталог)
  - `GET /niches/{slug}/download?token=X` (только для имеющих право)
  - `POST /niches/{slug}/install` (вебхук от инстанса студента)
- Лицензирование: каждый `.niche` файл подписан HMAC-ключом, чтобы исключить пиратство

### CLI `sl-claw` (новый command)
- `sl-claw niche install <file_or_url>` — установка
- `sl-claw niche export <slug>` — выгрузка
- `sl-claw niche list` — что установлено
- `sl-claw niche update <slug>` — обновление до новой версии

### Веб-кабинет на `token.lux-promo.com`
- /cabinet/niches — витрина «мои ниши» + «доступные»
- /marketplace — публичный каталог
- /marketplace/niches/{slug} — страница ниши с описанием, кейсами, ценой

### Версионирование ниш
- Семантическое версионирование: 1.2.3
- Major: ломающие изменения (новый формат meta.yaml)
- Minor: новые объекции, новые сценарии
- Patch: исправления
