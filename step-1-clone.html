Шаг 1. Получите ядро репозитория

Клонируйте проект с GitHub одной командой. Это полное production-ready ядро — тот же код, который работает в проде на sl-sclaw.coreviaflow.space.

Что вам потребуется

  • Git установленный в системе (проверка: git --version)
  • Любой терминал (Terminal, iTerm, Warp на Mac; PowerShell или WSL на Windows)
Если Git ещё не установлен

На macOS — выполните в терминале xcode-select --install. На Ubuntu/Debian — sudo apt install git. На Windows — скачайте с git-scm.com.

Клонирование

Откройте терминал, перейдите в папку, где будете хранить проекты, и выполните:

git clone https://github.com/vimana-tcg/AI_sales_write.git
cd AI_sales_write

Если вам выдадут SSH-команду — используйте её только если у вас настроен SSH-ключ для GitHub. Иначе HTTPS-вариант выше работает у всех.

Что внутри

После клонирования вы увидите такую структуру:

AI_sales_write/
├── app/
│   ├── api/         # FastAPI — единая точка входа всех каналов
│   ├── core/        # LangGraph агент, стадии воронки, промпты
│   ├── channels/    # Web, Telegram, WhatsApp, Instagram, Voice
│   ├── tools/       # RAG, оплата, CRM, Calendar, Handoff
│   ├── memory/      # mem0 — долговременная память клиента
│   └── db/          # SQLAlchemy модели
├── scripts/         # Загрузка каталога и обучение
├── docker/          # Dockerfile + Postgres init
├── Ниши/            # Knowledge-база (.md файлы)
├── docker-compose.yml
├── pyproject.toml   # Зависимости (Poetry)
├── .env.example     # Шаблон конфигурации
└── README.md

Главные файлы, которые мы будем трогать

Файл Что в нём Когда правим
.env Все ключи и настройки Шаги 3–4
app/core/tenants.py Persona вашей ниши Шаг 6
app/core/prompts.py Системный промпт продавца Когда захотите кастомизировать стиль
app/core/stages.py 7 стадий воронки Если хотите сменить методологию
scripts/ingest_catalog.py Загрузка вашего каталога в RAG Шаг 6

Откройте проект в редакторе

Если у вас Cursor или VS Code — откройте папку проекта прямо из терминала:

# Cursor
cursor .

# VS Code
code .

Если команда не работает — откройте редактор вручную и выберите папку AI_sales_write через File → Open Folder.

Проверка

Вы должны видеть структуру проекта в боковой панели редактора и быть в папке AI_sales_write в терминале.

Если возникли проблемы

  • git: command not found — Git не установлен. См. блок выше.
  • Permission denied (publickey) — вы используете SSH-вариант без настроенного ключа. Используйте HTTPS: git clone https://github.com/...
  • fatal: repository not found — проверьте URL, проверьте что вы залогинены если репо приватный
  • Нужен доступ к репо — напишите нам на hello@coreviaflow.space
