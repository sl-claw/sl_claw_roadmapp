Уроки

M1.1 Что такое AI-продавец и как он работает 8 мин 🟡 в разработке
Цель: студент за 8 мин понимает категорию (sales-движок, не «GPT-обёртка») и видит wow-демо.
Сценарий: face cam hook → слайд архитектуры → скринкаст рабочего бота через 5 каналов → таблица AI vs ChatGPT vs менеджер → кейсы +30% конверсии.
Студент: смотрит, в конце записывает 1 свою нишу.
Результат: wow + мотивация продолжать.
→ M1.2: «Ставим базовые инструменты — Python, Docker, VS Code.»
M1.2 Подготовка машины (Python, Docker, VS Code) 12 мин 🟡 в разработке
Цель: студент устанавливает 4 программы и видит зелёный «healthy» в Docker.
Сценарий: Mac (Homebrew + python@3.12) → Windows (WSL2 + apt) → Docker Desktop → VS Code + 2 расширения → Poetry → проверка `python3 --version && docker --version && poetry --version`.
Студент: параллельно ставит всё. Финальная проверка показывает 3.12.x.
Результат: 4 программы установлены, терминал реагирует на команды.
→ M1.3: клонирование репо.
M1.3 Клонирование sl-claw/core с GitHub 6 мин 🟡 в разработке
Цель: студент клонирует репо, открывает в VS Code, понимает структуру.
Сценарий: face cam → открываем github.com/sl-claw/core → `git clone` → `cd core` → `code .` → обзор папок (app/, scripts/, docker/) → ключевые файлы.
Студент: папка `core/` на машине, открыта в VS Code.
→ M1.4: получаем lux-token.
M1.4 Получение lux-token (один токен ко всем AI) 7 мин 🟡 в разработке
Цель: студент получает lux-token с балансом $5 и сохраняет его.
Сценарий: token.lux-promo.com → регистрация → пополнение карта/LiqPay/USDT → API Keys → Create → имя `sl-claw-dev` → копируем в заметки.
Студент: токен `lux-…` на руках, баланс $5.
→ M1.5: главный wow-урок — первый запуск.
M1.5 ★ .env + первый запуск (главный wow-урок) 15 мин 🌟 ключевой
Цель: студент видит как Anna отвечает в чате на его команды. Это wow-момент тарифа.
Сценарий: `cp .env.example .env` → вставляем lux-token и persona → `docker compose up -d postgres redis` → `poetry install` → `poetry run uvicorn …` → /health → WS-клиент → «Привет» → AI отвечает!
Студент: бот работает локально, получил wow.
→ M1.6: «бот отвечает обобщённо, теперь научим знать ВАШ каталог».
★ 6 Покупаем VPS + передаём Claude управление 10 мин 🌟 ключевой
Цель: у Claude SSH-доступ к VPS. Дальше всё через диалог. Все будущие данные клиентов будут жить на этом VPS.
Сценарий: покупка Contabo Cloud VPS NVMe (€5/мес) или Hetzner CPX21 (€7/мес) → передача IP+пароль Claude → Claude настраивает SSH-ключи и отключает password auth.
Студент: покупает VPS (5 минут руками — единственный шаг). Передаёт credentials Claude.
Результат: Claude управляет VPS. SSH-ключи безопасные.
→ Урок 7: Claude разворачивает бота.
★ 7 Claude разворачивает бота на VPS 24/7 12 мин 🌟 ключевой wow MINI
Цель: бот работает 24/7. Postgres + Redis + mem0 на VPS — все данные у студента.
Сценарий: один промпт «разверни моего бота на VPS». Claude по SSH: Docker → git clone → docker compose up → ждёт healthcheck. Backup БД настроен на Backblaze B2.
Студент: один промпт. Закрывает ноут. Пишет в TG → бот отвечает.
Результат: 🎯 бот в проде 24/7. Все данные на VPS студента. Backup ежедневный.
→ Урок 8: загружаем каталог в работающий прод-бот.
M1.6 Загрузка вашего каталога в RAG 12 мин 🟡 в разработке
Цель: студент заливает 5–10 своих товаров и видит как AI их называет.
Сценарий: формат demo_catalog.json → студент готовит 5 товаров → `python scripts/ingest_catalog.py my.json` → «Сколько стоит товар X?» → AI достаёт из каталога.
Студент: бот знает каталог, отвечает с реальными ценами.
→ M1.7: «Это уже круто. Но дальше — лимиты.»
