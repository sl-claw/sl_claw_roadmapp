Это не курс — это услуга

В отличие от MINI/CORE/PRO здесь нет уроков. Студент платит — команда работает за него. Документ ниже — рабочие SOP (standard operating procedures), по которым работает команда внедрения.

Что разрабатываем (рабочие процессы команды)

  • Notion-шаблон аудита бизнеса (15 вопросов, 5 блоков)
  • Чеклист анализа продукта и ЦА
  • SOP по разработке скрипта продаж под нишу
  • SOP по загрузке базы знаний (URL → crawl → PDF → Vision)
  • SOP по подключению канала клиента
  • Сценарий 60-минутной обучающей встречи с командой клиента
  • SLA-договор + чеклист передачи
  • 14-дневный план сопровождения
  • Внутреннее CRM для отслеживания клиентов (Notion / KeyCRM)
  • Создание team clients-implementation в GitHub с private fork под клиента

Что получает клиент

  • 🔓 Подключение к 1–2 каналам коммуникации
  • 🔓 Загруженная база знаний (каталог + сайт + PDF)
  • 🔓 Обучение команды клиента (60-минутный созвон)

  • 🔓 Опциональный месячный support-контракт от $300/мес

SOP I1 — Onboarding клиента после оплаты

Длительность процесса: 1–2 рабочих дня

  1. Webhook от Lemon Squeezy / LiqPay получает payment.success
  2. Access-service создаёт private GitHub fork ядра под брендом клиента
  3. Аккаунт-менеджер (АМ) звонит клиенту в течение 4 часов
  4. АМ высылает Notion-форму аудита (15 вопросов)
  5. Клиент заполняет за 1 рабочий день
  6. АМ + CTO разбирают ответы → составляют ТЗ

Артефакты: Notion-template аудита, email-шаблон welcome + Calendly, Slack-канал #client-{name}.

SOP I2 — Аудит бизнеса клиента

Длительность: 60-минутный созвон + 1 день анализа

15 вопросов по 5 блокам:

  1. Продукт: что продаёте, ценовая линейка, маржа, сезонность
  2. Клиент: ICP, главные боли, путь покупки, средний чек
  3. Каналы: где сейчас приходят лиды, конверсия по каналам
  4. Команда: сколько менеджеров, нагрузка, главные узкие места
  5. Цели на 3 мес: сколько лидов хотим, какая конверсия, какой ROI

Артефакты: заполненная Notion-форма, transcript созвона.

SOP I3 — Разработка скрипта продаж

Длительность: 2–3 рабочих дня

  1. CTO готовит TenantPersona на основе аудита
  2. Sales-coach пишет 5 типовых сценариев (новый лид, возражение «дорого», «подумаю», горячий, остывший)
  3. Загружаем 50+ возражений из objections_core.md + кастомизация под клиента
  4. Тестируем диалоги в Discord-канале, корректируем

Артефакты: persona.py, objections.md, spin_questions.md, test_dialogs.json.

SOP I4 — Загрузка базы знаний

Длительность: 1–2 рабочих дня

  1. Получаем от клиента: ссылку на сайт, прайс-лист (PDF), каталог (CSV/Excel), фото товаров
  2. Запускаем /setdemo URL для автоматического crawl
  3. Дополнительно bulk_train.py для objections.md + cases.md
  4. Vision-обработка фото товаров
  5. QA: 20 тестовых вопросов клиента → проверяем точность ответов

Артефакты: RAG-база заполнена, лог обучения, отчёт QA.

SOP I5 — Подключение каналов клиента

Длительность: 1 рабочий день

  1. Клиент даёт доступы (Telegram BotFather, Meta App, домен сайта)
  2. CTO настраивает webhook'и через Coolify
  3. Тестируем диалоги в каждом канале (минимум 3 проверки в каждом)
  4. Включаем мониторинг через Uptime Kuma

Артефакты: работающие каналы, мониторинг, документация для клиента.

SOP I6 — Запуск и обучение команды

Длительность: 60-минутный созвон с командой клиента

  1. Демо рабочего бота в каждом канале
  2. Обучение работе с админ-командами (/settings, /train, /answer)
  3. Передача доступов: Coolify, GitHub fork, Discord канал
  4. Документация в Notion: «как тренировать вашего AI-продавца»

Артефакты: запись обучающего созвона, передача доступов, чеклист.

SOP I7 — 14 дней сопровождения

Ежедневные задачи команды:

  • Утро: проверка логов LangSmith за вечер/ночь
  • Обработка unknown questions через /answer
  • Коррекция диалогов в которых AI ошибся
  • Еженедельный созвон 30 мин: показываем метрики, тюним промпт

Метрики, которые мониторим:

  • Conversion rate по стадиям 1→7
  • Среднее время ответа
  • % unknown questions
  • NPS клиентов

SOP I8 — Передача в support и upsell

По истечении 14 дней:

  1. Финальный отчёт: метрики, что сработало, что улучшить
  2. Рекомендация: переход на ежемесячный support-контракт ($300/мес — лёгкий, $800/мес — полный)
  3. Если клиент готов — подписание support-договора
  4. Если не готов — система продолжает работать, клиент сам себе саппорт
